Store all fetched data in database

Implement a background scheduler:

Run ONLY 2 times per day:
Morning (08:00)
Evening (20:00)

Fetch:
Matches (fixtures) for today and next 2–3 days
Matches (results) from last 3–5 days
Competition standings if needed
Use Football-Data.org endpoints:
/matches (dateFrom & dateTo filters)
/competitions (for league data)
Limit API usage:
Maximum 10 API calls per run
Maximum 20 calls/day
DATABASE AS CACHE

Ensure:

/fixtures returns stored upcoming matches
/results returns past matches
/predictions returns stored predictions

NO endpoint should call external API directly.

DATABASE RULES
Use match_id as unique key
Avoid duplicate inserts
Update existing matches when status changes (scheduled → finished)

home_win probability
draw probability
away_win probability
over 2.5 goals
BTTS

high_confidence if probability ≥ 0.7

RESULTS TRACKING

Store:
won
lost
